# AGENTS.md

This file provides guidance to coding agents when working with code in this repository.

## Project Overview

**eclpy** is a Python library that provides bindings for evaluating Common Lisp code through an Embeddable Common Lisp (ECL) runtime packaged as WebAssembly. It enables bidirectional evaluation between Python and Lisp, allowing Python to evaluate Lisp forms and vice versa.

**Purpose**: Bridge Python and Common Lisp through a WebAssembly runtime
**Language**: Python (3.14+), C, Lisp
**Key Technology**: Wasmtime + Emscripten + ECL compiled to WASM

## Repository Structure

### Core Package: `eclpy/`
The main Python package with layered architecture:

- **`__init__.py`** - Public API exports
- **`lisp.py`** - High-level `Lisp` facade and reference lifecycle management
- **`session.py`** - Low-level `EclSession` for Wasmtime lifecycle and WASM interaction
- **`proxy.py`** - Pythonic package proxy (`Package`, `_CallableSymbol`) for calling Lisp functions
- **`syntax.py`** - Fluent API (`L.expr()`, `L.quote()`, `L.array()`, etc.) for building expressions
- **`sexp.py`** - S-expression tree building (`SExp` class and subclasses)
- **`encode.py`** - Python value conversion to Lisp syntax
- **`protocol.py`** - Object-shaped JSON value protocol (bidirectional Lisp/Python conversion)
- **`objects.py`** - Python classes for Lisp values: `Symbol`, `List`, `Cons`, `Reference`
- **`hostenv.py`** - Emscripten `env` imports: file I/O, stat, Python eval/exec
- **`wasmmem.py`** - WASM linear-memory helpers and WASI error codes
- **`errors.py`** - `EclError` exception class
- **`__main__.py`** - CLI entry point (`eclpy` command): REPL, `-e EXPR`, file runner
- **`runtime_lisp.py`** - Source constant for Lisp helper package
- **`runtime.lisp`** - Lisp-side helper package (serialize/deserialize, py-eval, py-exec, ASDF bridge)
- **`ecl_eval.wasm`** - Pre-built WASM binary (generated by build script)
- **`asdf.lisp`** - ASDF source bundled in wheel (copied from vendored ECL)
- **`swank/`** - SWANK server source; `backend.lisp`/`ecl.lisp`/`gray.lisp`/`match.lisp`/`packages.lisp`/`rpc.lisp`/`swank-core.lisp`/`swank-repl.lisp` are copied unmodified from `vendor/slime` by the build script (gitignored, regenerated on each build), and `loader.lisp` is a project-authored no-compile loader shim (tracked in git). Loaded on demand by `ecl-python:start-swank`.

## Architecture & Design

### Value Crossing the Python-Lisp Boundary
The bridge uses a **single object-shaped JSON protocol** for all values:
1. Python value → `protocol.to_protocol()` → JSON → UTF-8 bytes
2. Bytes → WASM memory shuttle
3. JSON → `ecl-python:json-decode` → `ecl-python:deserialize` → Lisp value
4. **Reverse**: Lisp value → `ecl-python:serialize` → `ecl-python:json-encode` → JSON → Python

**JSON Protocol Envelopes**:
- Success: `{"protocol": "eclpy", "version": 1, "status": "ok", "value": {...}}`
- Error: `{"protocol": "eclpy", "version": 1, "status": "error", "condition_type": "...", "message": "..."}`

**Value Types**: `nil`, `true`, `int`, `ratio`, `float`, `string`, `symbol`, `list`, `dotted-list`, `vector`, `package`, `ref`

### WASM Runtime Features
- **Emscripten 6.0.1** with WebAssembly exception handling (`-sSUPPORT_LONGJMP=wasm -sWASM_LEGACY_EXCEPTIONS=0`)
- Native `setjmp`/`longjmp` lowered to WASM exceptions (avoids JS trampolines on every Lisp call)
- Wasmtime configured with `wasm_exceptions`, `wasm_function_references`, `wasm_gc` enabled
- WASI Preview 1 (WASI 0.3 component model not supported by local toolchain)

### Layered API Design
1. **Syntax API** (`syntax.py`): Aggressive Python→Lisp conversion for everyday use
2. **Strict SExp API** (`sexp.py`, `encode.py`): Explicit tree building for precise control
3. **Raw Source** (`SExp.raw()`): Embed raw Lisp when needed
4. **Package Proxies** (`proxy.py`): Pythonic attribute access to Lisp symbols
5. **Low-level** (`session.py`): String-based evaluation and raw memory management

### Reference Lifecycle
- Lisp objects that cannot cross the boundary (functions, packages, closures) become `Reference` handles
- References are stored in a hash table on the Lisp side with numeric IDs
- Python side owns the handle lifetime (release with `context manager` or `close()`)
- Automatic cleanup on `Lisp` session exit

## Key Files & Important Functions

### `lisp.py`
- **`Lisp.__init__()`** - Initialize session, load Lisp helpers, set ASDF source path
- **`Lisp.eval(form: SExp) → Any`** - Primary API: evaluate explicit SExp
- **`Lisp.start_swank(port=4005, dont_close=True) → None`** - Blocking SWANK/SLIME server; bypasses the JSON evaluation protocol (calls `session.eval` directly) so unhandled request errors reach ECL's native condition system instead of `ecl-python:evaluate`'s `handler-case`. Run from a background thread.
- **`Lisp.close()`** - Release references and close WASM session
- **`Lisp._eval_sexp()`** - Internal: wrap in lambda, call helper, decode result
- **`Lisp._make_reference()` / `_release_reference()`** - Reference lifecycle

### `session.py`
- **`EclSession.__init__()`** - Load WASM module, initialize wasmtime, call `eclpy_init`
- **`EclSession.eval(code: str) → str`** - Evaluate and print result
- **`EclSession.eval_json(code: str) → str`** - Evaluate and return JSON
- **`_engine_config()`** - Enable WASM exceptions, function refs, GC
- **`_resolve_wasm_path()`** - Resolve from `wasm_path`, `ECL_WASM` env, or default

### `protocol.py`
- **`decode_result(node, lisp) → Any`** - Decode top-level success/error envelope
- **`decode_value(node, lisp) → Any`** - Recursively decode value nodes
- **`to_protocol(value) → dict`** - Convert Python value to protocol dict
- **`dump_value(value) → str`** - Encode Python value as JSON text

### `encode.py`
- **`to_syntax_api_expr(value) → SExp`** - Convert Python value to Lisp expression (handles strings as symbols, tuples as lists)
- **`to_syntax_api_literal(value) → SExp`** - Convert to literal (quotes lists)
- **`to_data_expr(value) → SExp`** - Build Lisp code that reconstructs data (uses LIST, CONS, VECTOR)

### `proxy.py`
- **`Package`** - Dataclass proxy for a Lisp package
- **`Package.__getattr__()`** - Look up symbol, convert underscores to hyphens, check operator aliases
- **`Package._lookup()`** - Call `ecl-python:lookup-symbol`, handle missing/callable/value/symbol results
- **`find_package(lisp, name) → Package`** - Get Python view of a package
- **`_CallableSymbol.__call__()`** - Build argument list and call via `Lisp._eval_sexp()`

### `hostenv.py`
- **`define_emscripten_imports()`** - Register all `env` function imports
- **`env_import(name, has_result, py_globals) → callback`** - Create host function
- **`eval_python()` / `exec_python()`** - Evaluate/execute Python code from Lisp side
- **`read_host_file()`** - Load file content into WASM memory
- **`stat_host_file()`** - Return file kind (0/1/2) and mtime
- **`getcwd()` / `chdir()`** - Minimal syscall stubs for Emscripten startup
- **`socket_listen()` / `socket_accept()` / `socket_poll()`** - Non-blocking listen/accept/readability-check host callbacks backing `sb-bsd-sockets` server-side socket ops (alongside the pre-existing client-side `socket_resolve`/`socket_connect`/`socket_send`/`socket_recv`/`socket_close`)

### `native/eclpy_eval.c`
- **`eclpy_init()`** - Boot ECL, load runtime.lisp, set up symbol cache
- **`eclpy_eval(source_ptr, source_len) → output_ptr`** - Parse forms, evaluate, print last result
- **`eclpy_eval_json(source_ptr, source_len) → output_ptr`** - Parse forms, evaluate, serialize to JSON
- **`eclpy_alloc(size)` / `eclpy_free(ptr)`** - Memory allocation in WASM
- **`eclpy_last_error()`** - Return current error message
- **`eclpy_socket_listen_wrapper()` / `eclpy_socket_accept_wrapper()` / `eclpy_socket_poll_wrapper()`** - C wrappers exposing host TCP listen/accept/non-blocking-poll to Lisp as `ecl-python::%socket-listen`/`%socket-accept`/`%socket-poll`

### `eclpy/runtime.lisp`
- **`ecl-python:evaluate(fn)`** - Core evaluation: call lambda with no args, serialize result
- **`ecl-python:serialize(value) → plist`** - Convert Lisp value to protocol plist
- **`ecl-python:json-encode(plist) → string`** - Encode plist to JSON
- **`ecl-python:json-decode(string) → plist`** - Decode JSON to plist
- **`ecl-python:deserialize(plist) → value`** - Reconstruct Lisp value from plist
- **`ecl-python:py-eval(source)` / `py-exec(source)`** - Evaluate/execute Python from Lisp
- **`ecl-python:lookup-symbol(package-name, symbol-name) → lookup-result-plist`** - Package introspection
- **`ecl-python:value(id) / release-object(id)`** - Reference lifecycle
- **ASDF file bridge**: `load`, `probe-file`, `truename`, `file-write-date` shimmed to read host files
- **`sb-bsd-sockets` package**: minimal `socket`/`inet-socket` classes and `socket-bind`/`socket-listen`/`socket-accept`/`socket-name`/`sockopt-reuse-address` backed by the `%socket-*` host bridge primitives, matching the real API the vendored `swank/ecl.lisp` drives unmodified
- **`ecl-python:start-swank(&key port dont-close)`** - Loads bundled SWANK source from `*swank-source-directory*` (once per session), patches `swank-compile-string`/`swank-compile-file` to EVAL source instead of `compile-file` (no native compiler in the sandbox), replaces `swank-debugger-hook` to abort to top level instead of entering SLDB (its backtrace walk over `SI::IHS-TOP`/`SI::FRS-TOP` traps the WASM instance), then calls `SWANK:CREATE-SERVER` with `:style nil` (`:communication-style nil`, no real threads in this build). Blocks the calling thread.

## Build & Development Workflow

### Prerequisites
- Python 3.14+
- `uv` package manager
- Emscripten 6.0.1 (for building WASM)
- GCC/Clang (for native host ECL)

### Building WASM Runtime
```bash
uv run python scripts/build_ecl_wasm.py
```
This:
1. Builds host ECL (native, for cross-compilation tools)
2. Configures ECL for `wasm32-unknown-emscripten` with exception handling flags
3. Links `native/eclpy_eval.c` into WASM
4. Smoke-tests via `EclSession()`
5. Copies result to `eclpy/ecl_eval.wasm`
6. Copies ASDF source to `eclpy/asdf.lisp`

### Testing
```bash
uv run python -m unittest discover -s tests
uv run coverage run -m unittest discover -s tests
uv run coverage report -m  # Enforces 100% coverage
```

Use `unittest` only. Do not add or rely on `pytest`; the project intentionally has no pytest configuration or dev dependency.

### Linting & Type Checking
```bash
uv run ruff check .
uv run basedpyright
```

`basedpyright` checks both `eclpy/` and `scripts/`, so build-hook imports such as `hatchling` must remain available in the dev dependency group.

### Building Distributions
```bash
uv build --wheel --out-dir dist --clear
uv build --sdist --out-dir dist --clear
```
Requires `eclpy/ecl_eval.wasm` and `eclpy/asdf.lisp` to exist first. `runtime.lisp` is also explicitly included in both wheel and sdist metadata.

After packaging changes, inspect both artifacts. The sdist must not contain `.claude/`, `.codex/`, `.venv/`, `build/`, `dist/`, or cache directories.

### Test Coverage
- **100% enforced** via `pyproject.toml` configuration
- Covers: raw evaluation, strict SExp, syntax API, proxies, macros, bidirectional eval, protocol conversion, cons/list, higher-order functions, references, ASDF, error paths

### Running the CLI
```bash
uv run eclpy                     # interactive REPL (eclpy-user package)
uv run eclpy -e "(+ 1 2)"        # evaluate expression
uv run eclpy path/to/script.lisp # run file
```

### Testing Against Local WASM
```bash
ECL_WASM=/path/to/custom.wasm uv run python -m unittest discover -s tests
```

- WASM exceptions (not JS trampolines)
- WASI Preview 1 (not WASI 0.3)
- 100% test coverage enforcement

## Security Model

**Not a sandbox.** Designed for trusted local code only:
- `SExp.raw()` embeds arbitrary Lisp source
- `ecl-python:py-eval` / `py-exec` run in host Python with normal privileges
- ASDF/file bridge reads ordinary host paths
- Do not expose APIs to untrusted input without application-level guards

## Notable Design Decisions

1. **Single JSON protocol**: Unified representation avoids parallel codec bugs
2. **Type-restricted `Lisp.eval()`**: Forces explicit SExp usage, catches user errors early
3. **Native WASM exceptions**: Trades build complexity for ~100x faster calls vs JS trampolines
4. **Reference IDs**: Handles objects that cannot cross the boundary (functions, streams)
5. **ASDF via file bridge**: Practical source-only module loading without full native environment
6. **100% coverage enforced**: Catches regressions in core bridge logic
7. **SWANK via source EVAL, no SLDB**: Bundles unmodified upstream SWANK/SLIME source; compiles via EVAL instead of `compile-file` (no C compiler in the WASM sandbox) and replaces the debugger hook to abort instead of entering the interactive debugger (backtrace stack-walking traps the WASM instance, not a catchable condition)
